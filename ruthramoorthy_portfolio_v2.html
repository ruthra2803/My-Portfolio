Brain-Controlled Smart Wheelchair
EEG-based mobility assistance with eye-movement navigation, obstacle avoidance, GPS tracking, and emergency alerts.
ESP32EEG SensorsGPSWireless Comm.

The Problem

Millions of people with motor disabilities — including those with ALS, spinal cord injuries, or muscular dystrophy — cannot operate conventional wheelchairs. Traditional joystick-based systems are inaccessible to those who have lost limb control. The challenge: how do you give someone full mobility using only their brain and eyes?

The Solution

This system uses EEG (electroencephalogram) signals captured from the user's brain activity, combined with eye-movement tracking to determine directional intent. The wheelchair responds to these neural signals in real time — no hands required.

How It Works

  • EEG headset captures brain activity and classifies movement intent using signal processing
  • Eye-movement sensors detect gaze direction for navigational control (left, right, forward, stop)
  • ESP32 microcontroller processes inputs and drives motors via wireless communication
  • Ultrasonic sensors provide real-time obstacle detection and automatic avoidance
  • GPS module tracks location and triggers emergency alerts if the user is unresponsive

Tech Stack

  • Microcontroller: ESP32 (dual-core, WiFi/BT enabled)
  • Sensing: EEG electrodes, IR eye-tracking sensors, ultrasonic HC-SR04
  • Communication: Wireless RF / BLE for real-time control
  • Navigation: GPS NEO-6M module with GSM alert system

Impact

This project directly addresses assistive technology for people with severe disabilities. It represents the intersection of neuroscience, embedded systems, and compassionate engineering — giving independence back to those who need it most.

Table Bot "MICKY" – AI Interactive Robot
Voice-controlled AI robot with ChatGPT integration, STT/TTS, and OLED facial animations.
ESP32ChatGPT APIEmbedded AIOLED

The Idea

MICKY started as a question: can a tiny ESP32-based robot hold an actual intelligent conversation, express emotions, and feel alive? The goal was to build a desktop companion that's more than a gimmick — a genuinely useful AI assistant in a physical, expressive form.

What MICKY Does

  • Listens to your voice using a microphone and converts speech to text (STT)
  • Sends your query to the ChatGPT API and receives an intelligent response
  • Converts the response back to speech (TTS) and speaks it aloud
  • Displays animated facial expressions on an OLED screen — happy, thinking, surprised
  • Reacts to the conversation context with matching emotions

Technical Challenges

Running AI on an embedded device is no small feat. The key challenge was handling API calls, audio processing, and OLED animations simultaneously on limited hardware. This required careful task scheduling and memory management on the ESP32.

Tech Stack

  • Brain: ESP32 with FreeRTOS task scheduling
  • AI: OpenAI ChatGPT API for intelligent responses
  • Audio: Microphone module for STT, speaker for TTS output
  • Display: 128x64 OLED with custom animation frames

Wireless Gesture-Controlled Robotic Arm
4-DOF robotic arm with trainable gesture control — your hand movements mirror the arm in real time wirelessly.
Servo MotorsMPU6050Microcontroller

Overview

A 4 Degrees-of-Freedom robotic arm that mirrors your hand and wrist movements in real time — no buttons, no joystick. Just natural gestures. Built for applications in remote manipulation, industrial automation, and assistive robotics.

How the Gesture Control Works

  • MPU6050 IMU sensor strapped to the user's hand captures 3-axis acceleration and gyroscope data
  • Motion data is processed and mapped to servo motor angles for each joint of the arm
  • Wireless transmission sends control signals from the glove to the arm in real time
  • The system is trainable — specific gestures can be recorded and mapped to custom arm positions

Mechanical Design

The arm features 4 independently controlled joints — shoulder, elbow, wrist rotation, and gripper — giving it a full range of motion. Servo motors at each joint provide smooth, precise movement with minimal latency.

Applications

This project has direct applications in bomb disposal, deep-sea exploration, surgical robotics, and accessibility tools for people with limited mobility in one hand.

LiFi-Based Hybrid Communication System
Laser-based high-speed data transmission with WiFi fail-safe and live OLED status display.
ESP32LDRLaser ModuleOLED

What is LiFi?

LiFi (Light Fidelity) is a wireless communication technology that uses light instead of radio waves to transmit data. It's potentially 100x faster than WiFi and immune to electromagnetic interference — making it ideal for sensitive environments like hospitals and aircraft.

This Implementation

  • A laser module transmits data by modulating light pulses at high frequency
  • An LDR (Light Dependent Resistor) on the receiver decodes the light pulses back into data
  • ESP32 handles both encoding/decoding and manages the hybrid fallback to WiFi if the laser link is interrupted
  • An OLED display shows real-time transmission status, signal strength, and active channel

Why Hybrid?

Pure LiFi fails if the light path is obstructed. By adding a WiFi fail-safe, this system ensures uninterrupted communication — automatically switching channels without user intervention. This makes it practical for real-world deployment.

Accident Detection System
ML-based accident detection on Raspberry Pi with automatic emergency alerts and GPS tracking.
Raspberry PiMachine LearningGPSPython

The Problem

Every year, thousands of accident victims die not from the crash itself, but from delayed emergency response. If help could be automatically dispatched the moment an accident occurs — regardless of whether the victim is conscious — survival rates would dramatically improve.

How It Works

  • Accelerometer and gyroscope sensors continuously monitor vehicle motion patterns
  • A trained ML model on Raspberry Pi classifies sudden impact events vs normal driving
  • On positive detection, GPS coordinates are captured and an emergency SMS/call is triggered automatically
  • The system can distinguish between hard braking, speed bumps, and actual collisions

ML Approach

Sensor data was collected across different driving conditions and used to train a classification model. The Raspberry Pi runs inference locally — no cloud needed — ensuring it works even without internet connectivity at the accident site.

Recognition

This project was presented at the Uyir Road Safety Hackathon 2025, where it reached the finalist stage — validating its real-world potential for saving lives on Indian roads.

ESP32 Air Mouse
Gesture-based wireless mouse using MPU6050 — control your cursor with wrist movements, no surface needed.
ESP32MPU6050IMU Sensors

Overview

A completely wireless, surface-free mouse that tracks your wrist orientation using a 6-axis IMU sensor and translates movement into cursor control. Perfect for presentations, smart TV control, or just showing off embedded systems skills.

Technical Details

  • MPU6050 captures 3-axis gyroscope + accelerometer data at high sample rate
  • ESP32 processes the IMU data, applies a complementary filter for smooth output
  • HID (Human Interface Device) protocol over Bluetooth — appears as a standard mouse to any device
  • Sleep mode activates after inactivity, extending battery life significantly

Power Optimization

One of the key engineering challenges was battery life. Using ESP32's deep sleep modes, the device wakes only when motion is detected, reducing idle power draw from ~160mA to under 10µA — a 16,000x improvement.
